# TODO LIST

- [ ] **[MEDIUM]** Surface a generated draft that has not been looked at
  - Type: feature
  - Description: Generate dispatches to Actions and takes roughly a minute, so the user closes the task and does something else. When the derived entry lands in the description, nothing announces it — the task renders identically to an idea that was never touched, and the draft is found only by reopening the task on a hunch. REVIEW and ASKING both reach the user through the badge word on mobile rows; a landed draft is the third state genuinely blocked on a human and is the only one still silent. Give it the same treatment: a `⌁ DRAFTED` badge state, cleared once the task has been opened.
  - Behavior: A task whose linked `agent_queue` row is in `drafted` and whose description has received the derived text renders `⌁ DRAFTED` in amber, on mobile as well as desktop, joining REVIEW and ASKING as a blocked-on-you state. Opening the task's description editor clears it — the signal means "you have not looked at this yet", and looking is the whole of what it asks for, so no separate dismiss control. Tapping the badge opens the description editor, matching what ASKING already does. The state is per-task and does not clear for other generated tasks. Once cleared it does not return for that generation; a subsequent Generate on the same task sets it again.

    - Clearing on open belongs in `showDescEditorModal`, written through `listLogic` like every other mutation, followed by a `TODO_RUN_STATUS_EVENT` dispatch so the row's badge repaints behind the modal.
