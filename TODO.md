  - Description: Add a "Generate tasks" action to the Build-plan stage in the Conceive view that turns the plan you wrote into real todos in that project. Tapping it sends the Build-plan stage's text as the task source — plus the project's other stages (Why / Concept / Requirements / Design) as background context — to the in-app Claude (the existing Worker chat path), asking it to decompose the plan into short, well-scoped task titles. Tasks are derived only from the Build plan; the other stages inform scope and phrasing only, never invent tasks of their own. The reply opens a review modal listing the proposed tasks as a checklist (all checked by default). You uncheck any you don't want and hit "Add N tasks," which creates them as committed todos in the currently selected project. Tasks whose title already exists in the project are auto-skipped — shown greyed with an "in tasks" tag, unchecked, and excluded from the count — so re-running can't spawn duplicates. This is pure client: it reuses the exported `chatWithWorker` chat call and needs no Worker change. The modal is the review surface from the approved mockup (a focused, dismissible overlay).
  - Behavior:
    1. The Build-plan stage (the SDLC stage labeled "Build plan") gets a "Generate tasks" button in its section header, enabled only when that stage's body is non-empty. The other stages are optional context — they're included only when non-empty and never gate the button.
    2. Tapping it opens the review modal in a loading state and calls Claude with the Build-plan text as the task source and the other non-empty stages as context. On reply, the modal renders the proposed task titles as a checklist (checked by default). On a failed, empty, or unparseable reply, the modal shows an error with a retry — not a silent close.
    3. Each proposed title is compared (trimmed, case-insensitive) against the selected project's existing todo titles; matches render disabled/greyed with an "in tasks" tag, unchecked, and don't count toward the total.
    4. The footer shows "Add N tasks" where N is the live count of checked, non-duplicate items; it's disabled when N is 0. Cancel and the modal's standard dismissals (close button, backdrop, Escape) close without creating anything.
    5. On "Add N tasks," each checked non-duplicate task is created as a committed todo in the selected project through the normal add-todo path (same default due date / status / placement as typing a task), so each persists to localStorage and mirrors to Supabase. The modal then closes and the view switches to Projects so the new tasks are immediately visible.
