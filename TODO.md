## Bugs

- [x] **[MEDIUM]** Fix project row drag failing when clicking empty space after short titles
  - Description: On projects with short titles (e.g. "Project 1"), clicking in the empty area to the right of the title text and attempting a drag-reorder does nothing ‚Äî the drag never initiates. The entire project row should act as a drag handle regardless of title length. Root cause: `#projInput` uses `width: 100%` and fills the full `1fr` grid column of `#projChild`, so the visually-empty space after a short title is actually inside the input element. Once a user clicks an already-selected project (which sets `titleInput.style.pointerEvents = "auto"` in the project click handlers in `main.js`), that state is never reset ‚Äî the `blur` handlers only reset `cursor`, not `pointerEvents`. From then on, mousedowns in the empty area of the input are captured by the input instead of `#projChild`, blocking `dragstart`. Fix by either re-locking `pointerEvents` back to `"none"` on blur, switching to a CSS-class-driven edit mode so the default pointer-events state is reliable, or wrapping the title in a non-stretching element so `#projChild` owns the trailing empty space directly.
  - File: `toDoList_main/src/main.js`, `toDoList_main/src/style.css`
