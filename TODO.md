## Bugs

- [ ] **[MEDIUM]** Fix first-ArrowDown focus target on Today rows and add upward escape from Today/Calendar rows to header pills
  - Description: Two follow-up gaps in the keyboard nav extended to Today and Calendar (`todoRowKeyboardNav.test.js` companion work). (1) On the Today view, the first ArrowDown lands on the row's `.todayRowTitle` button rather than on the `.todayRow.todoRowCard` div itself, because `.todayRowTitle` is a native `<button>` and the browser's default focus walk reaches it before the row-level `tabindex="-1"` handler resolves "current row" from focus. The Today arrow-nav branch must detect this case — when `document.activeElement` is a descendant of a `.todayRow.todoRowCard` (rather than the row div itself), treat the enclosing row as the current row and redirect focus to the row div with `.todo-active` applied, the same nav-mode behavior committed Projects-view rows have. After this fix, the first ArrowDown into the Today view focuses the row div, second ArrowDown advances to the next row div — the title button is reachable via Enter or Tab, not as the default ArrowDown landing spot. The same fix applies symmetrically to Calendar `.calendarCell` buttons: if `activeElement` is the cell itself or a descendant, treat the cell as current rather than relying on focus matching the row directly. (2) On the Today and Calendar views there's no upward escape from the first row/cell back into the header chrome (PROJECTS / TODAY / CALENDAR pills + hamburger + pomodoro / music / settings on the right). On the Projects view the corresponding escape uses the sidebar transitions (`sideMain` ArrowUp → `sidebarToggle`, then nav handler walks the header pills via ArrowLeft/ArrowRight), but on mobile the sidebar is collapsed and on the Today/Calendar surfaces no equivalent boundary handler exists. Add an ArrowUp boundary in the Today arrow-nav branch that, when the focused row is the first navigable row in the today list container, focuses the active header pill (the TODAY pill — `#viewPillToday`) so the existing nav-level ArrowLeft/ArrowRight handler can then walk across all seven header controls without the user needing the mouse. Same boundary for the Calendar view top row of cells, jumping to `#viewPillCalendar`. Both transitions preventDefault and stopPropagation to keep the cross-pane ArrowLeft/ArrowRight handler from also firing. Tests pin both contracts: a descendant of the first row receives the ArrowDown keystroke and after the handler the row div is `document.activeElement` carrying `.todo-active`; an ArrowUp on the first Today row focuses `#viewPillToday`; an ArrowUp on a first-row Calendar cell focuses `#viewPillCalendar`.
  - File: `toDoList_main/src/main.js`, `toDoList_main/tests/todoRowKeyboardNav.test.js`, `toDoList_main/tests/headerFooterArrowKeyNav.test.js`
