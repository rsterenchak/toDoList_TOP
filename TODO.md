- [ ] **[MEDIUM]** Task row: generate the entry with triage instead of typing it
  - Type: feature
  - Description: The description panel is already the entry-authoring surface — you type the entry into `descInput` and Inject commits it to TODO.md. But having Claude write that entry only exists on the Agent board, so getting a derived entry means leaving the task you are looking at and working a different surface. Add a Generate action beside Inject in the description panel that dispatches the existing derive path against this task's title and the project's inject target, and lands the derived text back into the task's description for review. Generate never injects — the existing Inject button stays the only thing that commits, so the derived text is always read before it becomes an entry.
  - Behavior: A committed row in a project with a resolved inject target shows a Generate action in its description panel alongside Inject. Tapping it dispatches derive and puts the panel into a pending state: Generate is replaced by a non-blocking "Generating…" indicator, Inject is disabled for that row, and the description textarea becomes read-only so a derive landing cannot overwrite text typed mid-flight. Pending state is persisted, so navigating away, switching projects, or reloading and returning shows the row still generating rather than reverting to an idle button. On completion the derived text is written to the task's description through listLogic, the panel returns to idle with Inject enabled, and the textarea is editable — the user reviews and edits before injecting. On failure or give-up the panel returns to idle with the description untouched and a dismissible failure indicator. Generate is unavailable (hidden) when the project has no inject target, matching how the Inject button's no-target state already gates.
