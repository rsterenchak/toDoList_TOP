  - Architectural decisions (locked in via Phase 1 review and refined for Phase 6): (B1) When cloud has data and local also has data on first sign-in, cloud wins â€” the local cache is discarded and the normal hydration path runs. (B2a) Migration is gated by a per-user marker in localStorage (`migrated_user_<userId>: true`) so it runs once per user per device.
