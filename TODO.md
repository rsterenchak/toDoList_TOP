- [x] **[MEDIUM]** claudeSheet: attach a task to the chat so the whole conversation stays scoped to it — Completed: 2026-07-22
  - Type: feature
  - Description: `openChatWithSeed` pre-fills the composer with a task's title, description, and pending question, but that context is one-shot — it lives in the first message and is gone by the second, so a multi-turn conversation about a task loses track of which task it is. Add a durable attachment instead: a task can be attached to the chat session, renders as a scope chip above the composer, and rides along with every turn until it is detached or the session is reset. This is the difference between seeding a conversation and scoping one, and it is what makes chat usable for working a single task rather than only opening one.
  - Behavior: A committed task row offers a "Discuss" action that opens the Claude sheet with that task attached. The sheet shows a scope chip above the composer carrying the task title and a detach control; with nothing attached the chip reads as unscoped so the state is never ambiguous. While a task is attached, every message sent — Fast or Deep — carries the task's title and description as context, so follow-up turns need no re-explanation. Detaching clears the chip and stops the context riding along; the conversation itself is untouched. "New Chat" clears the attachment along with the transcript. The attachment survives closing and reopening the sheet within a session, so switching to Tasks to look at something and coming back does not silently drop scope. Attaching a different task replaces the current one — at most one task is attached at a time.
