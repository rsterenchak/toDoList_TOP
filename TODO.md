  - Type: feature
  - Description: The description panel is already the entry-authoring surface — you type the entry into `descInput` and Inject commits it to TODO.md. Add a Generate action beside Inject that flags the task for triage and lands the resulting draft back into the task's description for review. Generate never injects — Inject stays the only thing that commits, so derived text is always read before it becomes an entry. This supersedes an earlier attempt that no-changed because the landing path was left as an open question; that question is answered below.

Because this uses the existing queue, a generating task ALSO appears on the Agent board under In progress and then Drafted — that is intended, not a leak; one queue, one place work is tracked.   - Type: feature
  - Description: The mobile description-editor modal is the only detail surface a task has below 1024px — the on-row status badge is CSS-hidden at that breakpoint, so this modal is where an item is actually read and worked. It currently reads as a form (body-font sentence-case controls, no pipeline state at all) rather than as the instrument panel the rest of the pipeline design uses. Restructure it to lead with a read-only phase rail rendering `derivePhase`, move the controls to the SpaceMono uppercase treatment used across the pipeline surfaces, mark Generate as the dispatched action it is, and move the manual STATUS segmented control below the actions so the derived pipeline phase and the user's own annotation are no longer stacked adjacent and read as one control.
  - Behavior: Opening a task's description on mobile shows, in order: a header line carrying the task title (uppercase display) and the close control; a four-node rail — IDEA · DRAFT · REVIEW · DONE — with nodes before the current phase filled and the current one highlighted; the entry textarea under a THE ENTRY label; Generate with its dispatch mark and a line naming which budget it spends; Inject; the Clear / Copy pair; and finally the STATUS segmented control under a label distinguishing it from the rail. The rail is display-only and cannot be tapped to change phase — phase is derived from TODO.md, so there is nothing for a tap to mean. Renaming the task still works exactly as it does today from the header. Every existing control keeps its current behavior; only order, typography, and the added rail change.
